# ⚡ VibeDiff (バイブ・ディフ)

**AIエンジニア（バイブコーダー）のための、ロジック翻訳・差分解説ツール**

AI（Cursor, Cline, ChatGPT等）を使った開発で、「AIがどこをどう変えたか分からない」「動きの理由を言葉で理解したい」という不安を解消するために生まれた、ローカル完結型の差分解析ツールです。

## 🌟 なぜ VibeDiff なのか？ (Gitとの違い)

従来の `git diff` や開発ツールの差分表示は、エンジニア向けに「コードの変更」を伝えるためのものでした。
しかし、AIと並走する現代の開発では、**「コードがどう変わったか」よりも「アプリとしての役割がどう変わったか」**を知ることが重要です。

- **Git**: 「32行目に `fetch()` が追加されました」
- **VibeDiff**: 「外部のデータベースから最新情報を取得する **『窓口』** が作られました。これにより、画面にリアルタイムな情報が表示されるようになります」

VibeDiffは、コードを「意味（ロジック）」に翻訳する通訳者です。

## ✨ 特徴

- **プロジェクト種別の自動判別**:
  ファイル構成をAIが読み取り、「これはiOSアプリ」「これはReact Web」「これはPythonツール」と自動判定。それぞれの文脈に合わせた最適な解説を行います。
- **インテリジェント・スキップ**:
  `node_modules` や一時ファイルなど、人間が読む必要のない巨大なデータは賢くスキップ。必要な本質だけに集中できます。
- **完全ローカル動作**:
  File System Access APIを使用。ソースコードは一切サーバーに送信されず、あなたのブラウザ内だけで処理されるため、機密情報が含まれるプロジェクトでも安心です。

## 🛠 技術的な裏側 (Technical Stack)

このツール自体も、最先端の「サーバーレスWeb技術」で構築されています。

- **Frontend**: [React 18](https://react.dev/)
- **Logic Translation Engine**: 独自のパターンマッチングアルゴリズムにより、ソースコードからフレームワーク特有の動き（状態管理、通信、UI制御など）を抽出。
- **In-Browser Compiler**: [Babel standalone](https://babeljs.io/) を使用し、ビルド工程なしでJSX/Reactをブラウザ上で実行。
- **Diff Engine**: [jsdiff](https://github.com/kpdecker/jsdiff) を活用した高速な行単位比較。

## 🚀 使い方

1. [VibeDiff 公開ページ](https://oyajibuki.github.io/diff/) にアクセス。
2. **ステップ①（分析）**: 修正前のフォルダを選択。AIがプロジェクトの種類と構造を把握します。
3. **ステップ②（差分）**: 修正後のフォルダを選択。変更箇所がリストアップされます。
4. **解説を読む**: 左側のリストからファイルやフォルダ（スキップ済み項目含む）を選択。右側にその役割とロジックの解説が表示されます。
